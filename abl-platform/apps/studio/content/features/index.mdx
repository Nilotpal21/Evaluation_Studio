## Feature Status Lifecycle

Features progress through the following statuses:

- **PLANNED** — Spec exists, implementation not started
- **ALPHA** — Early implementation, breaking changes expected
- **BETA** — Feature-complete, undergoing hardening and testing
- **STABLE** — Production-ready, fully tested, documented

## SDLC Pipeline

Every non-trivial feature goes through a 4-phase pipeline before implementation begins:

1. **Feature Spec** (`/feature-spec`) — Problem, scope, requirements, user stories
2. **Test Spec** (`/test-spec`) — E2E and integration test scenarios
3. **HLD** (`/hld`) — High-level architecture and 12 concerns
4. **LLD** (`/lld`) — Phased implementation plan with exit criteria

Each phase includes auditor review loops with mandatory CRITICAL/HIGH finding resolution.
