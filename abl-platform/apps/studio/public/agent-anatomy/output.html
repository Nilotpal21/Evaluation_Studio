How Output Works
OUTPUT controls what the agent says and how it says it. TEMPLATES define reusable response formats with interpolation. MESSAGES override built-in system messages. SYSTEM_PROMPT replaces the auto-generated prompt entirely. Together they give full control over agent voice across chat, voice, Slack, WhatsApp, and more.

TEMPLATES

Named, reusable response templates with {{interpolation}}, multi-format variants (markdown, voice, html, slack), and optional ACTIONS for interactive buttons.

MESSAGES + SYSTEM_PROMPT

MESSAGES override built-in agent messages (errors, prompts, escalations). SYSTEM_PROMPT replaces the entire auto-generated system prompt for full control.

TEMPLATES — Named Response Templates
Basic Template Content with {{interpolation}} › 
TEMPLATES:
  claim_confirmation:
    content: |
      Claim {{claim_id}} submitted successfully.
      Status: {{claim_status}}.
      Expected resolution: {{resolution_date}}.

  booking_summary:
    content: |
      Your booking is confirmed!
      Hotel: {{hotel_name}}
      Check-in: {{checkin_date}}
      Check-out: {{checkout_date}}
      Guests: {{num_guests}}
Templates are rendered with Handlebars. Variables come from GATHER fields, tool results, and session state. Use content for the default format.
Multi-Format Templates 6 format variants per template › 
TEMPLATES:
  claim_confirmation:
    content: |
      Claim {{claim_id}} submitted. Status: {{claim_status}}.
    formats:
      markdown: |
        ## Claim Submitted
        **ID:** {{claim_id}}
        **Status:** {{claim_status}}
      voice: |
        Your claim has been submitted successfully.
        The reference number is {{claim_id}}.
      html: |
        <div class="claim-card"><h2>Claim {{claim_id}}</h2></div>
      slack: |
        :white_check_mark: *Claim {{claim_id}}* submitted
        Status: `{{claim_status}}`
      whatsapp: |
        *Claim {{claim_id}}* submitted
        Status: {{claim_status}}
      adaptiveCard: |
        {
          "type": "AdaptiveCard",
          "body": [{"type": "TextBlock", "text": "Claim {{claim_id}}"}]
        }
Format Channel Use Case
default Any Plain text fallback when no format matches
markdown Chat / Web Rich text with headers, bold, lists
html Email / Web embed Full HTML rendering
voice Phone / Voice Short, spoken-friendly text (no visual formatting)
slack Slack Slack mrkdwn with emoji shortcodes
whatsapp WhatsApp WhatsApp-flavored markdown
adaptiveCard Teams / Webchat Microsoft Adaptive Card JSON
The runtime selects the format based on the active channel. If the specific format is missing, it falls back to content (default). Voice format should avoid markdown symbols and keep sentences short.
ACTIONS in Templates Interactive buttons and quick replies › 
TEMPLATES:
  claim_next_steps:
    content: |
      Your claim {{claim_id}} is ready for review.
      What would you like to do next?
    ACTIONS:
      - label: "Upload Documents"
        action: upload_documents
        style: primary
      - label: "Check Status"
        action: check_claim_status
        style: secondary
      - label: "Talk to Agent"
        action: escalate_to_human
        style: outline

  rating_request:
    content: "How was your experience?"
    ACTIONS:
      - label: "Great"
        action: rate_positive
      - label: "Okay"
        action: rate_neutral
      - label: "Poor"
        action: rate_negative
ACTIONS render as buttons in web/chat channels and as numbered options in voice. The action value maps to a tool call or flow step transition. Styles: primary, secondary, outline, danger.
Handlebars Helpers Conditionals, loops, formatting › 
TEMPLATES:
  order_summary:
    content: |
      {{!-- Conditional --}}
      {{#if premium_member}}Thank you, Premium member!{{else}}Thank you!{{/if}}

      Your items:
      {{!-- Loop --}}
      {{#each items}}
      - {{this.name}}: ${{this.price}}
      {{/each}}

      Total: ${{total}}
      {{!-- Unless --}}
      {{#unless discount_applied}}No discounts applied.{{/unless}}
Handlebars supports {{#if}}, {{#each}}, {{#unless}}, and custom helpers. Variables are auto-escaped in HTML format. Use {{{triple}}} for raw HTML.
MESSAGES — Override Built-in Messages
Default Message Keys 10 built-in overrides › 
MESSAGES:
  error_default: "Oops! Something went wrong. Let me try again."
  constraint_blocked: "I can't do that due to our policy."
  gather_prompt: "I need a bit more information:"
  conversation_complete: "All done! Anything else?"
  escalation_format: "Connecting you with a specialist..."
  invalid_handoff: "Sorry, I can't transfer you to that department."
  self_handoff: "I'm already the right agent for this."
  tool_fallback_desc: "Processing your request..."
Key When Used Default
error_default Unhandled error during execution "I'm sorry, something went wrong."
constraint_blocked Guardrail blocks an action "I'm not able to do that."
gather_prompt Prefixed when asking for missing fields "I need some information:"
conversation_complete All goals met, conversation ending "Is there anything else I can help with?"
escalation_format Handing off to human agent "Let me connect you with a human agent."
invalid_handoff Requested handoff target doesn't exist "I can't transfer to that agent."
self_handoff Agent tries to hand off to itself "I'm already handling this."
tool_fallback_desc Tool running, shown as status "Working on it..."
Custom Message Keys Extensible with any key name › 
MESSAGES:
  # Built-in overrides
  error_default: "Oops! Something went wrong. Let me try again."

  # Custom keys — reference anywhere
  custom_greeting: "Welcome back, {{customer_name}}!"
  after_hours: "Our office is closed. Hours: Mon-Fri 9am-5pm EST."
  maintenance_notice: "We're doing scheduled maintenance. Back in {{eta}} minutes."
  premium_upsell: "Upgrade to Premium for priority support!"
Custom keys are stored alongside built-in keys. Reference them in FLOW steps with respond: messages.custom_greeting or in tool responses. Supports {{interpolation}} just like templates.

Messages resolve via chain: ABL-defined MESSAGES take highest priority. If not found, checks DB overrides (per-tenant). Then falls back to PromptCatalog defaults. This lets tenants customize messages without touching ABL.
SYSTEM_PROMPT — Full Prompt Override
SYSTEM_PROMPT Replaces auto-generated prompt › 
SYSTEM_PROMPT: |
  You are a claims specialist for ABC Insurance.
  Rules:
  1. Always verify policy number first
  2. Never approve claims over $50,000
  3. Be empathetic but professional
  4. If unsure, escalate to a supervisor
  5. Always summarize the claim before submitting

  Tone: Warm but concise. Use the customer's name.
  Language: English only. Simple, clear sentences.
SYSTEM_PROMPT replaces the entire auto-generated system prompt. You lose framework features like auto-injected tool descriptions, guardrail reminders, gather instructions, and persona synthesis. Use only when you need complete control.
What Auto-Gen Includes Features lost when overriding ›
Auto-Generated Section Purpose Lost on Override?
Persona synthesis Combines agent name, description, instructions Yes
Tool descriptions Tells LLM what tools are available and how to use them Yes
Guardrail reminders Inline constraint summaries for the LLM Yes
Gather instructions Tells LLM which fields to collect Yes
Flow context Current step, transitions, goals Yes
Channel adaptation Voice/chat-specific instructions Yes
Prefer using instructions (top-level) to add behavior guidance. The framework merges it into the auto-generated prompt. Reserve SYSTEM_PROMPT for cases where auto-gen doesn't work (e.g., strict compliance templates, legacy prompt migration).

instructions (recommended)

Merged into the auto-generated prompt. Keeps tool descriptions, guardrails, and gather context. Safe, additive.

SYSTEM_PROMPT (advanced)

Replaces everything. Full control but you must manually include tool usage rules, constraints, and field collection logic.

Runtime Behavior
Templates are rendered with Handlebars at response time. The runtime resolves the template by name, selects the format matching the active channel, interpolates variables from session state, and appends any ACTIONS as interactive elements.
Messages resolve via a priority chain: ABL MESSAGES block (highest) → DB override (per-tenant) → PromptCatalog fallback (framework default). Custom keys are stored in the same map and accessible from FLOW respond steps and tool responses.
SYSTEM_PROMPT, when set, completely replaces the auto-generated system prompt. The runtime skips persona synthesis, tool description injection, guardrail merging, and gather instruction generation. The raw string is sent as-is to the LLM.
Common Patterns
Multi-Channel Templates Same content, optimized per channel › 
TEMPLATES:
  appointment_confirmed:
    content: |
      Appointment confirmed for {{date}} at {{time}}.
    formats:
      markdown: |
        ## Appointment Confirmed
        **Date:** {{date}}
        **Time:** {{time}}
        **Location:** {{location}}
      voice: |
        Your appointment is confirmed for {{date}} at {{time}}.
        That's at {{location}}. See you then!
      slack: |
        :calendar: *Appointment Confirmed*
        Date: {{date}} | Time: {{time}} | Location: {{location}}
Voice format uses conversational phrasing, no visual formatting. Slack uses mrkdwn with emoji. Markdown uses headers and bold for scannability.
Voice-Optimized Responses Short, spoken-friendly text › 
TEMPLATES:
  claim_status_voice:
    content: |
      Claim {{claim_id}} status: {{status}}.
    formats:
      voice: |
        Your claim is currently {{status}}.
        {{#if estimated_date}}We expect it to be resolved by {{estimated_date}}.{{/if}}
        Would you like me to send you an email update?
Voice responses should: (1) avoid abbreviations and symbols, (2) keep sentences under 15 words, (3) use natural pauses with periods, (4) end with a clear call-to-action or question.
Branded Error Messages On-brand error handling › 
MESSAGES:
  error_default: "Hmm, I hit a snag. Let me try a different approach."
  constraint_blocked: "I'd love to help with that, but it's outside what I'm able to do. Can I help with something else?"
  escalation_format: "Let me bring in someone who can help you better. One moment!"
  conversation_complete: "Glad I could help! Have a wonderful day."
Match your brand voice. Friendly brands use informal language. Enterprise brands use professional tone. Always end blocked actions with an alternative offer.
Anti-Patterns

SYSTEM_PROMPT without tool descriptions

Overriding the system prompt without listing available tools. The LLM won't know it can call tools, breaking agent functionality.

No voice format for voice agents

Using markdown templates on a voice channel. The agent reads out "asterisk asterisk bold asterisk asterisk" instead of spoken text.

Hardcoded strings instead of MESSAGES

Putting error text directly in FLOW respond steps. Tenants can't customize them and they're scattered across the ABL.

Missing fallback content

Templates with only formats: and no content:. If the channel doesn't match any format, the response is empty.
