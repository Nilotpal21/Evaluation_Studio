What Templates Are
Templates are named, reusable response definitions with multi-channel formatting. A single template can render differently for web chat (markdown), mobile (adaptive cards), voice (SSML), Slack (Block Kit), and WhatsApp (interactive messages). Each template supports {{variable}} interpolation and interactive actions (buttons, selects). 
TEMPLATES:
  claim_confirmation:
    content: |
      Your claim {{claim_id}} has been submitted successfully.
      Status: {{claim_status}}
      Estimated processing time: 3-5 business days.

    formats:
      markdown: |
        ## Claim Submitted ✅
        **Claim ID:** {{claim_id}}
        **Status:** {{claim_status}}
        **Processing:** 3-5 business days

        ---
        _You'll receive email updates at {{customer_email}}._

      html: |
        <div class="claim-card">
          <h2>Claim {{claim_id}}</h2>
          <p class="status">{{claim_status}}</p>
        </div>

      voice:
        ssml: |
          <speak>
            Your claim <say-as interpret-as="characters">{{claim_id}}</say-as>
            has been submitted. <break time="300ms"/>
            You'll hear back in 3 to 5 business days.
          </speak>
        instructions: "Speak clearly and warmly. Emphasize the claim ID."
        plainText: "Your claim {{claim_id}} has been submitted."

      slack: |
        {
          "blocks": [
            {"type": "header", "text": {"type": "plain_text", "text": "Claim Submitted ✅"}},
            {"type": "section", "fields": [
              {"type": "mrkdwn", "text": "*ID:* {{claim_id}}"},
              {"type": "mrkdwn", "text": "*Status:* {{claim_status}}"}
            ]}
          ]
        }

      whatsapp: |
        {
          "type": "interactive",
          "body": {"text": "Claim {{claim_id}} submitted. Status: {{claim_status}}"},
          "action": {"buttons": [
            {"type": "reply", "reply": {"id": "track", "title": "Track Claim"}}
          ]}
        }

      adaptiveCard: |
        {
          "type": "AdaptiveCard",
          "body": [
            {"type": "TextBlock", "text": "Claim Submitted", "size": "large", "weight": "bolder"},
            {"type": "FactSet", "facts": [
              {"title": "Claim ID", "value": "{{claim_id}}"},
              {"title": "Status", "value": "{{claim_status}}"}
            ]}
          ]
        }

    actions:
      elements:
        - id: "track_claim"
          type: button
          label: "Track My Claim"
          value: "track"
        - id: "contact_support"
          type: button
          label: "Contact Support"
          value: "support"

Voice Channel — 3 Engine Types
Voice Config SSML + Instructions + Plain Text › 
voice:
  # W3C SSML — Google Cloud TTS, Azure, Amazon Polly
  ssml: |
    <speak>
      Your claim <say-as interpret-as="characters">{{claim_id}}</say-as>
      has been submitted.
      <break time="500ms"/>
      You'll hear back in <emphasis level="moderate">3 to 5</emphasis> business days.
    </speak>

  # Natural language instructions — OpenAI Realtime, Gemini Live
  instructions: "Speak warmly and clearly. Spell out the claim ID letter by letter. Pause before the timeline."

  # Plain text fallback — ElevenLabs, all engines
  plainText: "Your claim {{claim_id}} has been submitted. You'll hear back in 3 to 5 business days."
Field Engine Format
ssml Google TTS, Azure, Amazon Polly W3C SSML markup with tags like <break>, <emphasis>, <say-as>
instructions OpenAI Realtime, Gemini Live Natural language style instructions
plainText ElevenLabs, all engines (fallback) Voice-optimized plain text
Always provide plainText as a fallback. Not all voice engines support SSML. The runtime falls back: ssml → instructions → plainText → content.
Interactive Actions
TEMPLATES:
  room_selection:
    content: "Choose your room type:"
    actions:
      elements:
        # Buttons
        - id: "standard"
          type: button
          label: "Standard Room"
          value: "standard"
          description: "$120/night"
        - id: "deluxe"
          type: button
          label: "Deluxe Suite"
          value: "deluxe"
          description: "$280/night"

        # Dropdown select
        - id: "extras"
          type: select
          label: "Add extras"
          options:
            - id: "breakfast"
              label: "Breakfast (+$25)"
            - id: "parking"
              label: "Parking (+$15)"
            - id: "spa"
              label: "Spa access (+$40)"

        # Text input
        - id: "special"
          type: input
          label: "Special requests"
          inputType: text
          placeholder: "Any special needs?"
          required: false

      submitLabel: "Confirm Selection"
      submitId: "confirm_room"

Variable Interpolation
# Simple variables
content: "Hello {{customer_name}}, your claim {{claim_id}} is {{claim_status}}."

# Nested object access
content: "Hotel: {{selected_hotel.name}} at ${{selected_hotel.price}}/night"

# Handlebars helpers (conditionals)
content: |
  {{#if damage_amount}}
  Damage estimate: ${{damage_amount}}
  {{else}}
  No damage estimate provided yet.
  {{/if}}

# Loops
content: |
  Your options:
  {{#each hotels}}
  {{add @index 1}}. {{name}} - ${{price}}/night ({{rating}}★)
  {{/each}}
