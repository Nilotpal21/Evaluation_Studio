2. The Approach — Inline Rich Text Overlay
When the user clicks into a text-heavy field in the Monaco editor (or types / inside one), an inline rich text overlay appears — like Notion's block editor. It provides formatting, variable insertion, and Handlebars helpers. On save/close, the rich content is serialized back to valid YAML with | block scalar.
Two modes of rich text editing:
Mode When Features
LLM Instruction Mode PERSONA, GOAL, INSTRUCTIONS, SYSTEM_PROMPT, tool description Markdown formatting, bullet lists, numbered lists, headers, bold/italic. No variable insertion (LLM reads these as-is).
User Response Mode RESPOND, TEMPLATES, MESSAGES, PRESENT, ON_START RESPOND Everything in LLM mode PLUS: {{variable}} picker, {{#if}} conditionals, {{#each}} loops, channel-specific formatting.
3. YAML Safety — Round-Trip Without Breaking DSL
The rich editor must produce content that round-trips through YAML parsing without corruption. Key challenges: preserving {{var}} tokens, multi-line strings, special YAML characters (: # -), and indentation.
The safe serialization strategy:
# Rule 1: Multi-line content ALWAYS uses YAML block scalar "|"
PERSONA: |
  Professional and empathetic claims assistant.
  Uses clear language.

# Rule 2: Single-line content uses double quotes
RESPOND: "Hello! How can I help?"

# Rule 3: {{variables}} are preserved as-is inside | blocks
RESPOND: |
  Your claim {{claim_id}} has been submitted.
  Status: {{claim_status}}

# Rule 4: Handlebars helpers preserved verbatim
RESPOND: |
  {{#each hotels}}
  {{@index}}. {{name}} - ${{price}}/night
  {{/each}}

# Rule 5: Markdown inside | block is safe (YAML doesn't interpret it)
PERSONA: |
  ## Role
  You are a **claims specialist** with expertise in:
  - Insurance claims processing
  - Policy interpretation
  - Customer empathy
The | (literal block scalar) preserves everything verbatim — markdown, variables, special characters. The rich editor serializes to | for any multi-line content. Single-line uses quoted strings with escaped characters.
Serialization rules:
Content Serialization Example
Single line, no special chars Double quotes "Hello!"
Single line with {{var}} Double quotes "Hello {{name}}!"
Multi-line (any) Block scalar | |\n line1\n line2
Contains : or # Double quotes or | "Note: important"
Contains " (quote) Escaped or | |\n He said "hello"

7. When to Show Rich Text Editor
The rich text overlay activates when the user's cursor enters a text-heavy field in Monaco. Detection uses the same AST context detector.
Trigger Action
Click/cursor into PERSONA: | block Open LLM Instruction Mode overlay
Click/cursor into RESPOND: | block Open User Response Mode overlay
Click/cursor into INSTRUCTIONS: text block Open LLM Instruction Mode overlay
Click/cursor into TEMPLATES content: block Open User Response Mode overlay
Click/cursor into SYSTEM_PROMPT: | block Open LLM Instruction Mode overlay
Type / inside any rich text overlay Show text-specific slash commands (not DSL commands)
Type {{ inside User Response Mode Show variable picker
Press Escape or click outside Close overlay, serialize to YAML, update Monaco
The rich text overlay sits on top of the Monaco editor for the selected text block. When closed, it serializes the content back to valid YAML and updates the Monaco model. The user can always switch to raw YAML view.

# Serialization flow
Rich Text (WYSIWYG) → Markdown string → YAML block scalar (|)
  - Bold → **text**
  - Heading → ## text
  - List → - item
  - {{var}} chip → {{var}} literal
  - {{#if}} block → {{#if condition}}\ncontent\n{{/if}}

# Deserialization flow
YAML block scalar (|) → Markdown string → Rich Text (WYSIWYG)
  - **text** → Bold
  - ## text → Heading
  - {{var}} → Inline chip
  - {{#if}} → Block marker
9. Edge Cases & Safety
Edge Case Solution
User types raw YAML syntax in rich editor Escape : and # in output. Wrap in quotes if single-line.
{{var}} inside bold: **{{name}}** Preserve both — serialize as **{{name}}**
Nested Handlebars: {{#if}}{{#each}} Render as nested block markers. Serialize verbatim.
Empty multi-line field Serialize as empty string "", not |
User switches to YAML view mid-edit Serialize current state, update Monaco, show code.
Invalid Handlebars (unclosed {{#if}}) Warning in editor, but allow it (don't block save).
RTL text (Arabic) Rich editor respects dir="rtl". YAML serialization unaffected.
Indentation in | block Always indent 2 spaces from parent key. Consistent.
Trailing newlines Use | (clip) not |+ (keep) — trims trailing newlines.
