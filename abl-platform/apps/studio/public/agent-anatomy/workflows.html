AGENT: Claim_Intake
GOAL: "Collect claim information and submit"
PERSONA: "Empathetic claims assistant"

TOOLS:
  submit_claim(data: object) -> {claimId: string}
    description: "Submit insurance claim"

GATHER:
  policy_number:
    prompt: "Policy number?"
    type: string
    required: true
    validate: "matches(value, '^POL-[0-9]{6}$')"
  damage_amount:
    prompt: "Estimated damage?"
    type: number
    required: true

CONSTRAINTS:
  pre_submit:
    - REQUIRE: damage_amount < 50000
      ON_FAIL: "Claims over $50K need manager review."

COMPLETE:
  - WHEN: claim_submitted == true
    RESPOND: "Claim submitted! Ref: {{claim_id}}"
Form-like agents: insurance intake, booking, onboarding. Gathers data, validates with constraints, submits via tool, completes when done.

AGENT: Secure_Claims_Agent
GOAL: "Process claims with security and memory"

GUARDRAILS:
  pii_protection:
    kind: input
    check: not_matches_pattern(input, "\\b\\d{3}-\\d{2}-\\d{4}\\b")
    action: redact
  injection_guard:
    kind: input
    check: not_matches_pattern(input.lower(), "ignore previous instructions")
    action: block

MEMORY:
  SESSION:
    - claim_status: string = "pending"
  PERSISTENT:
    - user.claim_history
  REMEMBER:
    - WHEN: claim_type != null
      STORE: { value: "{{claim_type}}", target: "user.last_claim_type" }
  RECALL:
    - ON: session:start
      INSTRUCTION: "Check user's claim history."

# ... TOOLS, GATHER, COMPLETE as before
Production agents handling sensitive data. Adds input/output guardrails for PII and injection. Memory for cross-session continuity.

AGENT: Hotel_Booking
GOAL: "Guide users through hotel booking"

ON_START:
  CALL: load_customer_profile
  RESPOND: "Welcome! Let's find you a hotel."

TOOLS:
  search_hotels(...) -> {hotels: array}
  create_booking(...) -> {booking_id: string}

FLOW:
  steps: [welcome, collect, search, confirm]

  welcome:
    REASONING: false
    RESPOND: "Where and when would you like to stay?"
    THEN: collect

  collect:
    REASONING: false
    GATHER:
      - destination: required
      - checkin_date: required
        type: date
    CORRECTIONS: true
    DIGRESSIONS:
      - INTENT: "cancel"
        GOTO: welcome
    COMPLETE_WHEN: destination AND checkin_date
    THEN: search

  search:
    REASONING: true
    GOAL: "Find best hotels matching criteria"
    AVAILABLE_TOOLS: [search_hotels]
    EXIT_WHEN: hotel_selected == true
    MAX_TURNS: 5
    THEN: confirm

  confirm:
    REASONING: false
    CALL: create_booking(hotel_id, guest_name)
    ON_SUCCESS:
      RESPOND: "Booked! ID: {{booking_id}}"
      THEN: COMPLETE
    ON_FAIL:
      THEN: search

GUARDRAILS:
  pii_guard:
    kind: input
    check: not_matches_pattern(input, "\\b\\d{3}-\\d{2}-\\d{4}\\b")
    action: redact

ON_ERROR:
  - type: tool_timeout
    retry: 2
    respond: "Retrying..."

COMPLETE:
  - WHEN: booking_confirmed == true
Structured multi-step processes: booking, onboarding, claims filing. Mixed mode with scripted collection + LLM-powered search. Digressions handle off-topic.

# ━━━ SUPERVISOR ━━━
SUPERVISOR: Insurance_Hub
GOAL: "Route customer requests to the right specialist"

HANDOFF:
  - TO: Claims_Agent
    WHEN: "claim filing, damage report, incident"
    PRIORITY: 1
    CONTEXT:
      pass: ["customer_id"]
      history: auto
    RETURN: true
  - TO: Billing_Agent
    WHEN: "payment, invoice, billing"
    PRIORITY: 2
    RETURN: true
  - TO: Policy_Agent
    WHEN: "policy details, coverage, terms"
    PRIORITY: 3
    RETURN: true

ESCALATE:
  triggers:
    - WHEN: "complex issue spanning multiple domains"
      reason: "Multi-domain request"
      priority: high

# ━━━ SPECIALIST (Claims) ━━━
AGENT: Claims_Agent
GOAL: "Process insurance claims"

DELEGATE:
  - AGENT: Policy_Lookup
    WHEN: "need to verify policy"
    INPUT: { policyId: "{{policy_number}}" }
    RETURNS: { policyData: "result.policy" }
    TIMEOUT: "30s"

HANDOFF:
  - TO: Billing_Agent
    WHEN: "billing question during claim"
    RETURN: true

GUARDRAILS:
  pii_guard: { kind: both, check: "...", action: redact }

MEMORY:
  PERSISTENT:
    - user.claim_history
  REMEMBER:
    - WHEN: claim_submitted
      STORE: { value: "{{claim_id}}", target: "user.last_claim_id" }
Enterprise multi-agent systems: customer service hubs, insurance platforms, banking. Supervisor routes, specialists handle domains, delegates for sub-tasks, escalation for humans.

The complete pipeline for a production agent. Every construct has a specific place in this pipeline. Understanding this flow is the key to designing agents effectively.
Construct → Pipeline Mapping:
AGENT + GOAL + PERSONA → System prompt identity
EXECUTION → LLM model, temperature, timeouts
ON_START → Session initialization
GUARDRAILS (input) → Content filter before LLM
GATHER → Entity extraction from user message
CONSTRAINTS → Business rule validation
TOOLS → JSON Schema for LLM tool_use
FLOW → Step execution engine
HANDOFF / DELEGATE → Multi-agent thread management
GUARDRAILS (output) → Content filter after LLM
MEMORY (REMEMBER) → Persistent fact storage
COMPLETE → Runtime completion detection
ON_ERROR → Error recovery at any point
ESCALATE → Human handoff at any point
HOOKS → before/after each turn
