## Description

Description

In any flow agent that uses a REASONING step with EXIT*WHEN conditions, when the model invokes the \_set_context* tool to store collected values, the LLM call immediately following that tool invocation consistently fails or returns an empty response. The flow does not advance to the next step as expected.

Steps to Reproduce

    - Configure a flow agent with a REASONING step that has:

    	- EXIT_WHEN condition dependent on one or more context variables

    	- _set_context_ as an available tool (via AVAILABLE_TOOLS: &#91;__set_context__&#93;)




    - Start the agent and reach the REASONING step

    - Provide input that causes the model to call _set_context_ to store the required values

    - Observe the response immediately after the _set_context_ tool call completes

Expected Behaviour

After _set_context_ successfully stores the required values:

    - If EXIT_WHEN condition is now satisfied → agent transitions to the next step and responds accordingly

    - If EXIT_WHEN condition is not yet satisfied → reasoning loop continues and the next LLM call executes cleanly

Actual Behaviour

The agent returns one of the following instead of continuing:

    - Empty response: "The agent returned an empty response. This usually indicates a configuration issue. Check model resolution, credentials, or agent flow steps."

    - An error on the LLM call immediately following the _set_context_ tool invocation

The debug trace shows:

    - _set_context_ executes and stores values successfully

    - The subsequent model call errors or produces no output

    - No tool_call complete trace event is emitted for the _set_context_ invocation

    - OTEL metric for _set_context_ duration/success is not recorded

Impact

    - Any flow agent using REASONING steps with _set_context_ is affected

    - The agent cannot advance past the step where context values are being collected

    - Users are shown an error mid-conversation with no recovery path

    - Retrying may or may not succeed depending on session state at the point of failure

Root Cause (Identified from Code)

The reasoning executor runs an LLM↔tool loop. When _set_context_ is called inside that loop, the next API call is constructed without the preceding thinking block — causing the function_call without reasoning item error. The fix belongs in reasoning-executor.ts where subsequent LLM calls after a tool result need to preserve the thinking block from the prior turn.
