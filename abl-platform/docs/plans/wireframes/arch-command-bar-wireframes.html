Phrase Strategy — Side by Side

The two journeys use completely different phrase sets. Home phrases are inspirational and use-case driven (what can you build?). In-project phrases are operational and capability-driven (what can you do with what you have?).

Interaction Flow

State transitions and keyboard shortcuts.

STATE MACHINE
COLLAPSED ---[click | Cmd+K]---> FOCUSED
FOCUSED ---[type query]---> FOCUSED (filtered results)
FOCUSED ---[Enter on search result]---> COLLAPSED + navigate
FOCUSED ---[Enter on "Ask Arch" | Tab]---> CHAT
FOCUSED ---[Enter on quick action]---> CHAT (auto-send)
FOCUSED ---[Esc]---> COLLAPSED
CHAT ---[send message]---> CHAT (streaming)
CHAT ---[smart route trigger]---> REDIRECT banner
REDIRECT ---[click "Open Arch AI"]---> COLLAPSED + navigate to full chat
CHAT ---[click expand icon]---> COLLAPSED + transfer buffer + navigate
CHAT ---[Esc]---> COLLAPSED (buffer preserved)
CHAT ---[page navigation]---> CHAT (context + phrases update, buffer preserved)
KEYBOARD SHORTCUTS
Cmd+K Open / focus command bar Esc Close to collapsed (preserves chat buffer) Up/Down Navigate search results / quick actions Enter Select result / send message Tab Switch from search to "Ask Arch" mode Cmd+Shift+K Open full Arch AI view directly
ROTATING PHRASE LOGIC
1. Detect journey: projectId ? 'in-project' : 'home'
2. Load base phrase set (15 phrases per journey)
3. If in-project and on entity page, swap first 5 with page-specific phrases
4. If agent context, inject {agentName} into phrase templates
5. Cycle at 3.5s per phrase, smooth vertical slide transition
6. Stagger offset when multiple bars visible (home has projects grid + bar)
