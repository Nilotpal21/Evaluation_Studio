### 7. Chart Colors Hook -- `packages/design-tokens/src/chart-colors.ts`

**PASS with bug (P1)**

`useSyncExternalStore` usage is structurally correct for React 18:

- Separate `subscribeToThemeChanges` subscription function
- `getThemeSnapshot` returns a monotonically increasing version counter
- `getServerSnapshot` returns `0` for SSR safety
- `useMemo` correctly depends on `version` to recompute resolved colors

**Bug**: MutationObserver accumulation on subscribe/unsubscribe cycles (P1).

When all listeners unsubscribe and a new one subscribes later:

1. Old MutationObserver still active (never disconnected, `observer` is closure-local with no stored reference)
2. `listeners.size === 1` is true again, so a NEW MutationObserver is created
3. Both old and new observers fire the same `listeners.forEach(fn => fn())` callback
4. Result: double-firing of callbacks, and observer count grows with each full subscribe/unsubscribe cycle

**Fix**: Store the observer reference at module scope and disconnect it when `listeners.size === 0`.

## Issues

### P0 -- Blocks Docker Builds (CRITICAL)

**[P0-1] Missing Dockerfile COPY for `packages/design-tokens/package.json`**

`packages/design-tokens/` is a new workspace package and a dependency of `@agent-platform/admin-ui`. Every Dockerfile that copies `admin-ui/package.json` must also copy `design-tokens/package.json`, otherwise `pnpm install --frozen-lockfile` will fail with an unresolvable workspace dependency.

Affected Dockerfiles (6):

- `apps/admin/Dockerfile`
- `apps/runtime/Dockerfile`
- `apps/studio/Dockerfile`
- `apps/search-ai/Dockerfile`
- `apps/search-ai-runtime/Dockerfile`
- `apps/multimodal-service/Dockerfile`

**Fix**: Add `COPY packages/design-tokens/package.json packages/design-tokens/package.json` to each Dockerfile, near the `admin-ui` COPY line.

---

**[P0-2] Tailwind content path missing `packages/design-tokens/`**

`apps/admin/tailwind.config.ts` content paths:

```ts
content: ['./src/**/*.{js,ts,jsx,tsx}', '../../packages/admin-ui/src/**/*.{js,ts,jsx,tsx}'];
```

This does NOT include `../../packages/design-tokens/src/**/*.{js,ts,jsx,tsx}`. Tailwind JIT scans content paths to determine which utility classes to generate. If the design-tokens package is the only file referencing a class name (e.g., `bg-purple-muted`), Tailwind will purge it.

Currently mitigated by:

1. Hand-written utility classes in `globals.css` (lines 237-318)
2. Class names also appearing in scanned `admin-ui/src/` files

But this mitigation is fragile. If hand-written utilities are ever removed (they look redundant), classes only referenced in design-tokens will silently disappear.

**Fix**: Add `'../../packages/design-tokens/src/**/*.{js,ts,tsx}'` to the content array in:

- `apps/admin/tailwind.config.ts`
- `apps/studio/tailwind.config.js` (for future use)

### P1 -- Should Fix Before Pass 2

**[P1-1] MutationObserver leak in `chart-colors.ts`**

File: `packages/design-tokens/src/chart-colors.ts:118-137`

The `subscribeToThemeChanges` function creates a new MutationObserver when `listeners.size === 1` but never stores a reference to disconnect it. If all listeners unsubscribe and later resubscribe, a second observer is created. Both observers fire callbacks, causing double re-renders.

**Fix**: Store observer at module scope. Disconnect when last listener unsubscribes.

```ts
let observer: MutationObserver | null = null;

function subscribeToThemeChanges(callback: () => void): () => void {
  listeners.add(callback);
  if (listeners.size === 1 && typeof document !== 'undefined') {
    observer = new MutationObserver(() => {
      themeVersion++;
      listeners.forEach((fn) => fn());
    });
    observer.observe(document.documentElement, {
      attributes: true,
      attributeFilter: ['data-theme', 'class'],
    });
  }
  return () => {
    listeners.delete(callback);
    if (listeners.size === 0 && observer) {
      observer.disconnect();
      observer = null;
    }
  };
}
```

**[P1-2] `tokens.css` not imported -- `OVERLAY_BACKDROP_LIGHT` will silently fail**

File: `packages/design-tokens/src/tokens.css`

The `tokens.css` file defines `.bg-overlay\/40` (line 30-32) which is used by the exported `OVERLAY_BACKDROP_LIGHT` constant. But no app imports `tokens.css`. When `OVERLAY_BACKDROP_LIGHT` is used in a component, `bg-overlay/40` will produce no visual effect.

**Fix**: Either:

- (a) Add `@import '@agent-platform/design-tokens/css';` to each app's globals.css, or
- (b) Add `.bg-overlay\/40` definition to each app's globals.css alongside `.bg-overlay`, or
- (c) Mark `OVERLAY_BACKDROP_LIGHT` as `@internal` / add a JSDoc warning that apps must import tokens.css

---

**[P1-3] `chart-card.tsx` duplicates `SEMANTIC_CHART_COLORS`**

File: `packages/admin-ui/src/components/chart-card.tsx:8-16`

`CHART_COLORS` in chart-card.tsx is a manual duplicate of `SEMANTIC_CHART_COLORS` from design-tokens. The values are identical (`hsl(var(--accent))`, etc.) but maintained separately. This contradicts the centralization goal of the design-tokens package.

**Fix**: Replace the local `CHART_COLORS` with a re-export:

```ts
import { SEMANTIC_CHART_COLORS as CHART_COLORS } from '@agent-platform/design-tokens';
export { CHART_COLORS };
```
