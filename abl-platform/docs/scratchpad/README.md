# Scratchpad

A working area to plan new ideas, perform reviews, brainstorm features, and explore requirements before creating actionable plans, RFCs, or tickets.

## Purpose

- **Ideation**: Sketch out feature concepts and explore feasibility against the current platform
- **Gap analysis**: Compare what exists vs. what's needed for a proposed capability
- **Requirement refinement**: Turn rough ideas into concrete, scoped requirements
- **Review notes**: Capture architectural review feedback before formalizing

## Guidelines

- Files here are **living documents** -- update them as understanding evolves
- Once a scratchpad doc matures into an actionable plan, create an RFC or spec in the appropriate folder and link back
- Use descriptive filenames: `cross-channel-identity-recall.md`, not `idea-1.md`
- Include a status line at the top (`Status: exploring | refining | ready-for-rfc | archived`)
