## 1. Graph Architectures

| Graph Feature                                                                                      | ABL Equivalent                                                                                                                                               | Status        | Notes                                                                                                                                                                                                                                                                                          |
| -------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Supervisor** (`FlowType.SUPERVISOR`) — Central LLM supervisor orchestrates workers via tool-use  | `SUPERVISOR` agent type with `MODE: reasoning` + `ROUTING` rules. Supervisor uses `__handoff__` system tool to route to child agents.                        | ✅ Equivalent | ABL supervisors are declared in DSL with explicit routing rules, priorities, and conditions. Compiled to IR `RoutingConfig` with `rules[]`, `default_agent`, `intent_classification`. See `compiler.ts:176-194`, `schema.ts:910-939`.                                                          |
| **Delegation** (`FlowType.DELEGATION`) — Autonomous workers delegate to each other via event edges | `DELEGATE` construct on any agent. Sub-agent invoked synchronously with `INPUT`/`RETURNS` mapping. `HANDOFF` with `RETURN: true` for async-style delegation. | ⚡ Better     | ABL separates two patterns: `DELEGATE` (synchronous call-and-return with typed I/O mapping) and `HANDOFF` (control transfer with optional return). Graph system conflates both into event-edge-based routing. ABL's typed `INPUT`/`RETURNS` mapping is superior to unstructured state sharing. |
| **SingleAgent** (`FlowType.SINGLE_AGENT`) — Single agent with tools                                | Any agent without `HANDOFF`/`DELEGATE`/supervisor type. Can be `MODE: reasoning` or `MODE: scripted`.                                                        | ✅ Equivalent | Implicit in ABL — any standalone agent is single-agent. No explicit flag needed.                                                                                                                                                                                                               |
| **Custom** (`FlowType.CUSTOM`) — Undefined custom pattern                                          | `execution.pipeline_order` in IR allows custom execution phase ordering.                                                                                     | ✅ Equivalent | ABL supports configurable pipeline ordering per agent: `['constraints', 'reasoning', 'delegates', 'escalation', 'handoffs', 'completion']` can be reordered. See `executor.ts:273-276`.                                                                                                        |

## 2. Node Types

| Graph Feature                                                                                         | ABL Equivalent                                                                                                                                                          | Status        | Notes                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **LLM Model Node** — Provider config per agent (model, credentials, streaming, temperature)           | `execution.model` or `execution.operation_models` in agent IR. 5-level model resolution chain: Agent IR → Agent DB → Project DB → Tenant DB → System default.           | ⚡ Better     | ABL's model resolution is richer: per-operation-type model selection (extraction uses fast tier, reasoning uses powerful tier), complexity-based auto-selection, and tenant-level policies. Graph system requires explicit per-agent LLM node wiring. See `model-resolution.ts:27-102`, `model-selector.ts`.                                                                                                                                                                                                                                            |
| **Supervisor Agent Node** — Supervisor with name, prompt, tools, recursionLimit                       | `SUPERVISOR` declaration with `PERSONA`, `GOAL`, `LIMITATIONS`, `ROUTING` rules. | **Worker Agent Node** — Worker with name, prompt, tools, description                                  | `AGENT` declaration with `MODE`, `PERSONA`, `GOAL`, `TOOLS`, `GATHER`, `CONSTRAINTS`, etc. Compiled to `AgentIR`.                                                       | ✅ Equivalent | ABL agents are richer: they include `GATHER` (progressive field collection), `CONSTRAINTS`, `MEMORY`, `ESCALATE`, `COMPLETE`, `ON_ERROR` — all declarative. Graph workers rely on prompt engineering for these behaviors.                                                                                                                                                                                                                                                                                                                               |
| **Proxy Worker** — External API/A2A protocol adapter                                                  | Agent with `type: http` or `type: mcp` tools, reachable via `__handoff__`. Response flows back to supervisor via `RETURN: true`.                                        | ✅ Equivalent | Any ABL agent with HTTP/MCP tools can serve as a proxy worker. Handoff target resolution is name-based (no type restriction). `toolExecutor` is inherited from parent session. Return flow works — child response propagates back to supervisor conversation.                                                                                                                                                                                                                                                                                           |
| **Memory Node** — Checkpointing enablement (Redis for voice, MongoDB for regular)                     | `MEMORY` construct with `session` and `persistent` declarations. Checkpointing via `memory-checkpointer.ts` (in-memory) and `redis-checkpointer.ts` (Redis).            | ✅ Equivalent | ABL's memory is declarative: `session: [field1, field2]`, `persistent: [user.name]`, `recall: ON_START`, `remember: WHEN condition STORE value`. Fact Store provides cross-session persistence.                                                                                                                                                                                                                                                                                                                                                         |
| **Tool Nodes** — Various tool types (see Section 4)                                                   | `TOOLS` section in ABL with 4 binding types: `http`, `mcp`, `lambda`, `sandbox`. Plus system tools (`__handoff__`, `__delegate__`, `__complete__`, `__escalate__`).     | ✅ Equivalent | See Tool Types section below.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| **PreProcessor Node** — Input preprocessing, memory update, prompt/tool control before agent LLM call | `CONSTRAINTS: always` group + PII Guard hook + `before_turn` hooks (`set` to update context, `call` to execute tools, `respond` to short-circuit).                      | ⚠️ Partial    | **What works:** Constraints run every turn before LLM. `before_turn` hooks can update `state.context` via `set` action. PII detection built-in. `gatherProgress` values appear in system prompt. **What doesn't:** `state.memory.session` values are NOT surfaced in system prompt (Concern #9). Tools and system prompt are built from `agentIR` statically at execution start — no dynamic modification of available tools or prompt content before the LLM call. `grant_memory` from handoff is parsed but never applied to receiving agent's state. |
| **PostProcessor Node** — Output postprocessing after agent                                            | Trace scrubber for output sanitization. `COMPLETE` conditions for output validation. `CONSTRAINTS` can apply post-tool-call checks (`pre_change`, `pre_cancel` groups). | ✅ Equivalent | ABL's constraint groups (`pre_change`, `pre_cancel`, `always`) serve as pre/post processors. Output scrubbing happens via trace-scrubber.ts (PII, secrets, auth tokens).                                                                                                                                                                                                                                                                                                                                                                                |

## 3. Edge Types

| Graph Feature                                                                           | ABL Equivalent                                                                                                                                     | Status        | Notes                                                                                                                                                                                                                                                                        |
| --------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- | ------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **LLM → Agent edge** — Associates LLM provider with agent                               | Implicit — model resolved per agent via 5-level resolution chain. No explicit edge needed.                                                         | ⚡ Better     | ABL auto-resolves models. Graph requires manual LLM→Agent edge wiring per agent.                                                                                                                                                                                             |
| **Supervisor → Worker edge** — Orchestration hierarchy                                  | `ROUTING` rules in supervisor + `available_agents` list in IR. Runtime resolves via `__handoff__` tool.                                            | ✅ Equivalent | Hierarchy defined declaratively in supervisor's `HANDOFF` section.                                                                                                                                                                                                           |
| **Memory → Supervisor edge** — Conversation memory enablement                           | `MEMORY` construct directly on the agent. No edge required.                                                                                        | ✅ Equivalent | Memory is a property of the agent, not a separate node requiring wiring.                                                                                                                                                                                                     |
| **Event Edge** — Event-based routing between agents/tools with conditions               | `HANDOFF` with `WHEN` conditions + `DELEGATE` with `WHEN` conditions. Flow `ON_INPUT` branches for scripted mode.                                  | ✅ Equivalent | ABL uses conditional routing via WHEN expressions evaluated against state context. Scripted mode adds `ON_INPUT`, `ON_SUCCESS`, `ON_FAILURE` branching.                                                                                                                      |
| **PreProcessor Edge** — Input preprocessing link                                        | Implicit — constraints are part of agent definition, not separate wired edges.                                                                     | ✅ Equivalent | No edge needed. `CONSTRAINTS: always` group runs automatically before every agent turn.                                                                                                                                                                                      |
| **Runtime Conditional Edges** — START→Supervisor, Worker→Supervisor, Worker→END routing | Pipeline execution order: `constraints → reasoning → delegates → escalation → handoffs → completion`. Configurable via `execution.pipeline_order`. | ⚠️ Partial    | ABL uses a linear pipeline rather than a graph. The execution order is deterministic per turn. Multi-turn routing happens via handoff/delegate actions returned from the pipeline. No explicit graph-level conditional edges — routing is action-driven.                     |
| **Proxy → Supervisor edge** — Response routing mode (orchestrator vs direct)            | Proxy agent as `__handoff__` target with `RETURN: true`. Child agent's final response flows back to supervisor's conversation history.             | ✅ Equivalent | A proxy agent (agent with HTTP tools) can be a handoff target. With `RETURN: true`, the child's response is copied back to the parent supervisor's conversation (runtime-executor.ts:3530-3536). The supervisor then processes the proxy's response and decides next action. |

## 4. Tool Types

| **Error Handling Tools** — Tools tagged as error handlers             | `ON_ERROR` construct with handler definitions per error type. Supports `RETRY`, `THEN: ESCALATE\|HANDOFF\|CONTINUE\|COMPLETE`.                | ✅ Equivalent | ABL's error handling is declarative and richer: 10 error types classified automatically, configurable retry with delay, recovery actions, and per-error-type handlers. See `error-executor.ts`.                       |
| **returnDirect** — Tool output goes directly to user                  | Not supported. Tool results always return to the LLM for processing.                                                                          | ❌ Gap        | Would need a `return_direct: true` property on tool definitions to bypass LLM post-processing.                                                                                                                        |

## 11. Advanced Features

| Graph Feature                                                                                                                           | ABL Equivalent                                                                                                                                                                                                                                                                                        | Status        | Notes                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| --------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Filler Messages (Progress)** — Static/dynamic progress messages during execution                                                      | Not supported. `pendingMessages` in flow-executor accumulates intermediate messages but is not a progress system.                                                                                                                                                                                     | ❌ Gap        | Needs a `FILLER_MESSAGES` construct with static/dynamic modes. | **Content Variable Resolution** — `{{variableName}}` in ALL components (prompts, tool descriptions, tool params, headers, auth configs) | `interpolateMessage()` / `interpolateRichTemplate()` in `evaluator.ts` — supports `${var}`, `{{var}}`, `{{#if}}`, `{{#each}}`. NOT called in system prompts, tool descriptions, or tool parameters. | ⚠️ Partial    | **Where ABL interpolates:** constraint ON_FAIL messages, handoff target/summary, escalation messages, complete messages, flow RESPOND (with rich templates). **Where ABL does NOT:** system prompt (persona/goal/limitations), tool descriptions, tool parameter descriptions/defaults, MCP headers. MCP executor has no header config at all.                                                                                                                                                                                                                                                     |
| **A2A Protocol** — Agent-to-Agent communication with streaming                                                                          | `HANDOFF` (control transfer) + `DELEGATE` (call-and-return). JSON context passing. No streaming between agents. No external A2A protocol.                                                                                                                                                             | ⚠️ Partial    | ABL has rich inter-agent communication via HANDOFF/DELEGATE but no external A2A protocol standard. Communication is same-process, not network-based.                                                                                                                                                                                                                                                                                                                                                                                                                  |
| **Direct Invocation Modes** — Agent/tool invocation bypassing supervisor                                                                | Not supported. ## Features Where ABL Is Superior

| #   | Feature                              | ABL Advantage                                                                                                                                                                                                       |
| --- | ------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| S1  | **Declarative agent definition**     | ABL defines agents in DSL with `PERSONA`, `GOAL`, `GATHER`, `CONSTRAINTS`, `MEMORY`, `ESCALATE`, `COMPLETE`, `ON_ERROR` — all compiled to IR. Graph system requires manual node/edge wiring and prompt engineering. |
| S2  | **Typed delegation**                 | `DELEGATE` with `INPUT`/`RETURNS` mapping provides typed, enforceable contracts between agents. Graph's event-based delegation is unstructured.                                                                     |
| S3  | **Progressive field collection**     | `GATHER` with entity extraction LLM call + validation + defaults. Graph system has no equivalent — agents must prompt-engineer field collection.                                                                    |
| S4  | **Complexity-based model selection** | Auto-selects model tier (haiku/sonnet/opus) based on agent complexity and operation type. Graph requires manual LLM node per agent.                                                                                 |
| S5  | **Structured error handling**        | 10 classified error types with per-type handlers, retry with delay, and recovery actions. Graph has 4 error types with simpler retry.                                                                               |
| S6  | **Rich template engine**             | `{{#if}}`, `{{#each}}`, nested paths in action messages/flow RESPOND. Graph only has simple `{{var}}` substitution. Note: ABL's engine is richer but applied in fewer places — see G0 gap for universal resolution. |
| S7  | **Constraint groups**                | Named constraint groups (`pre_change`, `pre_cancel`, `always`) that act as contextual guardrails. Graph scanners are binary (input/output).                                                                         |
| S8  | **Execution phase pipeline**         | Configurable phase ordering (`constraints → reasoning → delegates → escalation → handoffs → completion`). Graph relies on fixed state machine.                                                                      |
| S9  | **Scripted flows**                   | `MODE: scripted` with deterministic state machine (FLOW steps, COLLECT, ON_INPUT branching, digressions). Graph has no equivalent — all agents are LLM-driven.                                                      |
| S10 | **Observability depth**              | 17 trace event types with full LLM raw request/response capture, span hierarchy, and Observatory UI.                                                                                                                |
