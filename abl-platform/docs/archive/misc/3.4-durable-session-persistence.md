# 3.4 Durable Session Persistence — Architecture & Implementation

> **Status**: Implemented
> **PR**: Part of `runtime-changes-v2` branch
> **Date**: 2026-03-05

---

## Problem

Sessions live in Redis with a 30-minute TTL. Any idle gap > 30 min destroys the session permanently. This blocks:

- Async flows (HITL, long-running tools, scheduled tasks)
- Multi-day conversations (booking flows, support tickets)
- Session forking for debugging alternative paths
- Context compaction for long conversations

## Architecture

### Append-Only Snapshot Store (Hot/Cold Tiering)

```
┌─────────────────────────────────────────────────────┐
│                    MongoDB (Durable)                 │
│                                                     │
│  session_states  — Session snapshot (compressed)     │
│    - stateData (gzip Buffer, encrypted)             │
│    - threads[] (per-thread compressed state)         │
│    - resolution keys, compaction metadata            │
│    - TTL: 7 days (configurable)                     │
└──────────────────────┬──────────────────────────────┘
                       │ write-through (fire-and-forget)
                       │ cold-restore on Redis miss
┌──────────────────────┴──────────────────────────────┐
│                    Redis (Hot Cache)                  │
│                                                     │
│  sess:{tid}:{id}     — Session HASH (30min TTL)     │
│  sess:{tid}:{id}:conv — Conversation LIST (window)  │
│  (unchanged interface — TieredSessionStore wraps)    │
└─────────────────────────────────────────────────────┘
```

### Key Design Decisions

| Decision          | Choice                                                                                             | Rationale                                    |
| ----------------- | -------------------------------------------------------------------------------------------------- | -------------------------------------------- |
| Cold store format | Compressed (gzip) JSON in MongoDB Buffer fields                                                    | 60-80% size reduction; decompression is ~1ms |
| Write strategy    | Fire-and-forget after Redis write                                                                  | Zero added latency on hot path               |
| Cold restore      | Load + rehydrate to Redis on miss                                                                  | Subsequent reads are hot                     |
| Encryption        | Top-level Buffers via encryptionPlugin; embedded thread Buffers rely on MongoDB at-rest encryption | Plugin limitation with embedded arrays       |
| Tenant isolation  | All repo methods accept tenantId; findOne({\_id, tenantId}) pattern                                | Never `findById` per platform invariants     |
| Compaction        | Per-thread, threshold-based, before LLM calls                                                      | Automatic; no admin trigger needed           |
| Fork              | Deep clone at thread boundary                                                                      | `structuredClone` for full independence      |

## Data Model

### `session_states` Collection

```typescript
interface ISessionState {
  _id: string; // Same as session.id
  tenantId: string; // Required for tenant isolation
  projectId: string;
  agentName: string;
  version: number; // Optimistic concurrency
  stateData: Buffer; // gzip(JSON) — all session state except per-thread data
  threads: ISessionStateThread[]; // Per-thread compressed snapshots
  activeThreadId: string;
  threadStack: string[]; // Thread IDs (string, not indices)
  headSeq: number; // Reserved for future message_log integration
  lastCompactionSeq: number; // Reserved
  compactionSummary?: string;
  pendingAsyncTasks: string[]; // Reserved for async resume
  irData?: Buffer; // Reserved for IR caching
  compilationData?: Buffer; // Reserved for compilation caching
  resolutionKeys: ISessionStateResolutionKey[];
  encryptedFields: string[];
  parentSessionId?: string; // Fork lineage
  forkPoint?: number; // Fork lineage
  expiresAt: Date; // MongoDB TTL index
  lastActivityAt: Date;
  createdAt: Date;
  updatedAt: Date;
}
```

**Indexes**: `{tenantId, _id}`, `{tenantId, projectId, lastActivityAt}`, `{tenantId, resolutionKeys.artifactHash}` (partial), `{expiresAt}` (TTL)

## Component Architecture

### 1. SessionStateRepo (`session-state-repo.ts`)

Low-level CRUD for `session_states` collection.

| Method                              | Tenant-scoped              | Description                      |
| ----------------------------------- | -------------------------- | -------------------------------- |
| `upsert(session)`                   | Yes (via session.tenantId) | Compress + store snapshot        |
| `load(sessionId, tenantId?)`        | Optional                   | Decompress + rebuild SessionData |
| `getVersion(sessionId, tenantId?)`  | Optional                   | Projection query `{version: 1}`  |
| `delete(sessionId, tenantId?)`      | Optional                   | Remove cold snapshot             |
| `touch(sessionId, tenantId?)`       | Optional                   | Extend expiresAt                 |
| `resolveByArtifact(tenantId, hash)` | Yes                        | Channel artifact → sessionId     |

### 2. TieredSessionStore (`tiered-session-store.ts`)

Decorates any `SessionStore` (Redis or Memory) with cold storage.

**Hot path (zero latency added)**:

```
create(session) → primary.create(session); persistToCold(session) // fire-and-forget
save(session)   → primary.save(session);   persistToCold(session) // fire-and-forget
load(sessionId) → primary.load(sessionId)                        // Redis hit → done
```

**Cold restore (Redis miss)**:

```
load(sessionId) → primary.load() = null
                → coldRepo.load(sessionId)
                → primary.create(cold)      // rehydrate to Redis
                → return cold
```

### 3. CompactionEngine (`compaction-engine.ts`)

Automatic per-thread context compaction before LLM calls.

**Trigger**: `estimateTokens(thread.conversationHistory) > contextWindow * threshold`

**Process**:

1. Split conversation: older half → compacted, recent half → kept
2. Generate summary (LLM `chatWithToolUse` with empty tools, or extractive fallback)
3. Rebuild: `[summary_system_message, ...recent_messages]`
4. Original messages in MongoDB `messages` collection untouched (audit trail)

**Config**: `SESSION_COMPACTION_ENABLED=true`, `SESSION_AUTO_COMPACT_THRESHOLD=0.8`

### 4. Session Fork (`session-operations.ts`)

Fork at thread boundaries for debugging / alternative flows.

**Process**:

1. `structuredClone` threads up to fork point
2. Create new SessionData with independent state
3. Store via `sessionService.store.create()` (TieredSessionStore handles cold)
4. Return `ForkResult` with new sessionId

**WebSocket**: `fork_session` message → `session_forked` response (with tenant auth check)
