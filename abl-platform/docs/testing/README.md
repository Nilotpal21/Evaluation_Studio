# Testing Results

> **Last Updated**: 2026-05-16
> **Purpose**: Index of all test specifications generated via the SDLC pipeline (`/test-spec`). Each file tracks E2E, integration, and unit test scenarios for one feature.

## How It Works

- **One doc per feature** in `docs/testing/<feature-name>.md`
- Each doc is updated with a new **Iteration** entry every time the feature is tested
- The "Current State" and "Health Dashboard" sections always reflect the latest truth
- The "Test Coverage Map" shows exactly what's been tested and what hasn't
- The "Iteration Log" preserves the full testing history with timestamps
