# SearchAI UX Experience Audit — Feel Testing

**Target**: https://agents-dev.kore.ai/
**Purpose**: Evaluate the product like a UX designer — does it feel right? Is it intuitive? Delightful?
**Mode**: OBSERVE AND DOCUMENT — rate each area, note friction points
**Executed**: 2026-03-23 (Pass 1 — live browser observation on `AI_Research_Library` KB)

---

## Rating Scale

| Score | Label      | Meaning                                              |
| ----- | ---------- | ---------------------------------------------------- |
| 5     | Delightful | Exceeds expectations, feels polished and intentional |
| 4     | Good       | Works well, minor polish opportunities               |
| 3     | Acceptable | Functional but feels unfinished or generic           |
| 2     | Friction   | Causes confusion, hesitation, or frustration         |
| 1     | Broken     | Unusable or severely misleading                      |
| —     | Not tested |                                                      |

## Additional Observations (Not In Scoring Matrix)

### Naming / Jargon Leakage

- `AI_Research_Library` shown with underscores in the page heading — KB names should be rendered as display names with spaces, or spaces should be required at creation time
- Breadcrumb shows raw `019d169b-e734-73d7-a793-658830c097f7` UUID before project context resolves
- `progressiveSummarization`, `questionSynthesis`, `noiseDetection`, `scopeClassification`, `knowledgeGraph`, `mapping_suggestion` — six feature names with no display name mapping
- `mapping_suggestion` uses snake_case while all others use camelCase — naming inconsistency in production data

### Data Accuracy Bugs (erode trust)

- KB header: "0 Documents" despite 7 documents visible in Data tab
- Browse preview: "0 documents" despite 5 indexed chunks
- Chunks tab: all show "Untitled" in Document column (source linkage bug)
- Sources: Default source shows "0" docs despite 5 indexed chunks referencing content from uploaded files

### Positive Patterns to Preserve

- The breadcrumb updates correctly when Intelligence sub-tabs are active (shows "Pipeline", "Fields", etc.)
- The "Connected to server" toast is subtle and informative without being intrusive
- The "1.0 KB", "983 B", "2.1 MB" file size formatting in Documents table is precise and correct
- Status indicator dots (green/red) are used consistently to mean "live/active" vs "error"
- "Last indexed 22/03/2026" in KB header gives temporal context for data freshness
