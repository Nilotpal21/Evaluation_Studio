Metric Definitions

NDCG@k (Normalized Discounted Cumulative Gain)
Measures ranking quality accounting for position. A result at rank 1 contributes more than rank 10. Range: [0, 1]. Higher is better.
DCG@k = sum(rel_i / log2(i+1)) for i=1..k
NDCG@k = DCG@k / IDCG@k
Recall@k
Fraction of all relevant documents that appear in the top-k results. Measures completeness of retrieval. Range: [0, 1].
Recall@k = |relevant in top-k| / |all relevant|
MRR (Mean Reciprocal Rank)
Average of 1/rank of the first relevant result across queries. Measures how quickly users find a relevant result. Range: [0, 1].
MRR = mean(1 / rank_first_relevant)
Precision@k
Fraction of top-k results that are relevant. Measures noise in retrieved context (critical for RAG). Range: [0, 1].
Precision@k = |relevant in top-k| / k
Hit Rate@k
Fraction of queries where at least one relevant document appears in top-k. Binary per-query measure. Range: [0, 1].
Hit Rate@k = |queries with >= 1 relevant in top-k| / |all queries|
RAGAS Faithfulness
Measures if generated claims are supported by retrieved context. Uses LLM-as-judge to extract claims and verify against source chunks. Range: [0, 1].
Faithfulness = |supported claims| / |total claims|

Recommended Evaluation Tools

RAGAS
RAG-specific evaluation framework. Faithfulness, answer relevance, context precision/recall. LLM-as-judge metrics.
pip install ragas
DeepEval
Pytest-native evaluation. Hallucination, toxicity, faithfulness metrics. CI/CD friendly with threshold assertions.
pip install deepeval
BEIR Benchmark
Standard IR benchmark suite. 18 datasets across diverse domains. NDCG@10 is primary metric. Published leaderboard for comparison.
pip install beir
MTEB (Massive Text Embedding Benchmark)
Embedding model evaluation across 56+ datasets and 8 tasks. Retrieval subset validates embedding quality. Hugging Face leaderboard.
pip install mteb
ranx
Python library for computing IR metrics (NDCG, MAP, MRR, Recall, Precision). Fast evaluation on large result sets. Fusion comparison utilities.
pip install ranx
trulens-eval
RAG triad evaluation (context relevance, groundedness, answer relevance). Dashboard for tracking metrics over time. Feedback functions.
pip install trulens-eval

Structured Data (Excel/CSV) Retrieval

CSV/Excel data uses a completely different retrieval path from unstructured documents. Rows are stored in ClickHouse as JSON strings (NOT in OpenSearch). Retrieval uses LLM-generated text-to-SQL with JSONExtract() on ClickHouse. Only 1 metadata chunk per table exists in OpenSearch for table discovery. Phase 13 tests cover this path.

Structured Data Retrieval Path (CSV/Excel) User Query: "What is total Q4 revenue by region?" | v StructuredDataQueryRouter.route() |-- analyzeIntent(): keyword heuristics score SQL vs semantic |-- tableDiscovery(): find relevant table(s) from metadata chunks | v TextToSQLService.generateSQL() |-- schema context: column names, types, descriptions, sample rows |-- LLM prompt: "Generate SELECT-only SQL. Use aggregate functions..." |-- validation: no destructive ops, must start with SELECT | v ClickHouse: queryWithLogicalSQL() |-- rewrite: Region -> JSONExtract(row_data, 'Region', 'String') |-- rewrite: Q4_Revenue -> JSONExtract(row_data, 'Q4_Revenue', 'Float64') |-- enforce: WHERE tenant_id = ? AND index_id = ? |-- execute: native ClickHouse query | v Results -> Agent LLM -> Natural language response KEY: Row data is JSON strings in ClickHouse, NOT vectors in OpenSearch KEY: No semantic search on CSV/Excel — always text-to-SQL KEY: No vocabulary generated — NL filter terms won't resolve
