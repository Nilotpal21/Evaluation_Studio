The problem today

Building multi-agent systems is YAML and prayers.

Teams spend weeks wrestling with prompt structure, tool definitions, routing logic, and brittle integration code. The blueprint drifts from the implementation. Nothing is the source of truth.

  • ×
    Hand-written YAML for every agent, every tool, every hand-off — easy to get wrong, hard to review.
  • ×
    Prompt drift. The system prompt, the docs, and the actual behavior diverge by week three.
  • ×
    No deterministic flow. The LLM decides when to advance, which tool to call, and whether to ask the user. Surprises in production.
  • ×
    Integration is the last mile. Tools, mock servers, and diagnostics all live in different places — or don't exist yet. 
# agent.yaml — by hand, hope it compiles
name: support_router
model: "claude-sonnet-4-5"
tools:
  - name: lookup_order
    type: http
    url: "https://api.example.com/orders/{id}"
    # wait, do we need auth? what shape?
routing:
  - when: "intent == 'refund'"     # typo: it's `refund_request`
    handoff: refunds_agent
  - when: "always"                  # this evaluates false :(
    handoff: generalist
gather:
  - field: order_id
    prompt: "What's your order ID?"
    # no on_fail, no on_input, no validation
on_complete: ???                       # is this a thing?

How it works

A coordinator that doesn't gamble.

The LLM is great at language, terrible at lifecycle. Arch AI splits the responsibility: the coordinator owns when to advance; the specialist owns what to say.

Layer 1

Coordinator

Deterministic phase machine. Validated state transitions. Scope classifier that knows the difference between a typo fix and a topology overhaul. The LLM never advances phases — it proposes, the coordinator decides.

↓
Layer 2

Specialists

11 expert personas, each with a focused system prompt and a curated tool set. The right specialist runs the right phase — onboarding asks the questions, the architect designs topology, the construct expert writes ABL.

↓
Layer 3

Knowledge & tools

35+ knowledge cards covering every ABL construct, workflow pattern, and platform limit — selected per turn within the prompt budget. Tool calls are validated before execution; client-side tools cleanly hand control to the user.

↓
Layer 4

Single source of truth

One markdown specification per project. One append-only journal of every decision, mutation, and validation. Resume any session and pick up exactly where you left off — no replaying transcripts.

What you ship

Six artifacts. All in sync.

Every Arch AI session produces the same set of outputs. The spec, the topology, and the code never disagree — they're generated from the same source.

📄

Spec document

A human-readable markdown specification. The single source of truth for the project — survives long after the chat is archived.

🌐

Topology

Validated multi-agent graph: agents, roles, hand-offs, build order. Generated via topological sort, not hand-drawn.

⚙️

Compiled agents

Each agent comes through skeleton → validate → auto-fix → compile. By the time you see it, it's already compilable ABL.

🛠

Mock server

A runnable mock for every external tool referenced in the topology. Demo end-to-end before integrating a single real API.

🩺

Diagnostics report

Typed findings with severity, category, and one-click fix templates. Never a vague "this might be wrong."

📜

Decision journal

Append-only timeline of every decision, mutation, validation. Audit-ready by construction. Resume any session, anywhere.
