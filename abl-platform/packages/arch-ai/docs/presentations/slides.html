How it stays safe

Six gates keep the loop sane

A general-purpose model left alone will drift, loop, or skip steps. Six deterministic gates make sure that doesn't happen.

1 Phase machine

Static edge table

Validates every transition. Only legal backward edge is BUILD → BLUEPRINT for large-scope mutations.

2 State machine

Atomic, gate-free

IDLE ⇄ ACTIVE → COMPLETE → ARCHIVED. Atomic findOneAndUpdate — no read-then-write races.

3 Scope classifier

Small vs Large

SMALL stays in phase. LARGE during BUILD triggers backtrack. diffTopology preserves work.

4 Build-gate queue

Pure derivation

pickNextGate(topology, files, approved) → next, needs_generation, or all_done.

5 Loop detection

SHA-256 fingerprint

Per-turn fingerprint of LLM output. Same response twice → guard_tripped, not burned tokens.

6 Executor guards

Pre-call checks

Max reinvocations, stall windows, cumulative time. Caught before round-trip.

Knowledge layer

35+ cards · selected per turn · within budget

The model never has to invent ABL syntax or platform limits. Every relevant rule is encoded as a card and injected on demand.

How selection works

  1. Phase + specialist define the eligible card pool
  2. Selector ranks cards by turn relevance
  3. Cards added in priority order until prompt budget hits
  4. PLATFORM_LIMITS_CARD always included — never skipped

What a card contains

  • Construct grammar (gather, handoff, flow…)
  • Real ABL examples that compile
  • Common mistakes & the right pattern
  • When NOT to use it
  • Cross-references to related cards

Why budget matters

  • Big prompts dilute attention
  • Big prompts explode cost & latency
  • Smallest relevant set keeps focus tight
  • Same prompt section every time — predictable
